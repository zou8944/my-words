### 23. [从 Antigravity 转 Claude Code 几天的心得](https://www.v2ex.com/t/1203411#reply31)

> 前几天由于 Antigravity 砍额度严重，我终于下决心转向目前地球上最强的 AI 编程助手，拥抱最纯的 opus token ，用了几天有些心得。
> 
> ### Cladue Code 比 Antigravity 好用的部份
> 
>   1. **Model 额度** ：这是提升最有感的部分，我开 100 刀的 max5x 方案，每天高强度用 5-6 小时 opus 用量都不超过 20%。
> 
>   2. **CLI 整合** ：cc 本身就是 cli 应用，调用 cli 没有任何权限问题；反重力要另外调命令行的时候总是不太听话。
> 
>   3. **写代码能力** ：同样是 opus 4.6 ，在 cc 里面调用的就是更会写代码，意图理解能力也强过反重力。在反重力里，我还单独写了一个 skills 让 agent 读项目文档理解上下文；在 cc 里只要让它读过几次，下次遇到类似的问题它就会自动去文档位置找来读。
> 
>   4. **Tools** ：cc 特有功能，光是网站搜索的能力就比反重力强，反重力甚至不懂怎么用 curl ，每次都要费勁巴拉打开瀏览器操作 DOM 。
> 
>   5. **Skills** ：虽然反重力也有 Skills ，但 cc 的 skills 比反重力更好用，不但生态更全，能力也更强（得力于 cc 的 Tools ）。例如我装了一个 /flutter-review 的 skills ，每次 opus 写完代码我就调用这个 skills 来 review 代码再修正，比我人力 prompt 还精确。
> 
>   6. **工作流** ：因为全程 cli ，大部分操作都可以用键盘完成，不用分心用鼠标、中断思路，突然有想法可以马上 `/btw` 补充，效率更高。
> 
> 
> 
> 
> ### Antigravity 比 Cladue Code 好用的部份
> 
>   1. **Implementaion Plan** ：反重力的 Plan Mode 会生成实施计划的 markdown ，可以直接像改论文一样在想批注的部份加注。cc 没那么好改，只能多花一些 prompt 说明想法，或是一开始就给足够约束的 prompt 。
>   2. **生成图片** ：例如我需要批量生成测试用户的头像並用脚本上传到服务器，cc 没法直接生成图片，反重力可以同时画图+写脚本。
>   3. **UI 绘制** ：因为 cc 也能上传图片，还有 ui 相关的 skills 可以帮助设计，这点我觉得反重力没多大优势；不过很多人反馈反重力做 UI 方便，也加上吧。
> 
> 
> 
> ### 结论
> 
> 后悔没早点开始用 Claude Code 。
