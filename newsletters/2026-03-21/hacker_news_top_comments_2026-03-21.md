### 11. [malisper在"Anthropic对OpenCode采取法律行动"中的新评论](https://news.ycombinator.com/item?id=47446104)
> 鉴于很多人对此事的意义有疑问，我来解释一下。
> 
> Anthropic 在这里涉及两个不同的产品：Claude API 和 Claude Code。Claude API 采用基于使用量的定价，用得多付得多。而 Claude Code 提供月度订阅，给予用户固定的使用额度。在生成同等数量令牌的情况下，订阅 Claude Code 要比使用 Claude API 便宜得多。
> 
> 对于像 OpenClaw 和 OpenCode 这样的第三方产品，Anthropic 已明确表示，这些产品应使用 Claude API，而非内部的 Claude Code API。然而，OpenClaw 和 OpenCode 一直在使用内部的 Claude Code API，因为当用户拥有 Claude Code 订阅时，内部 Claude Code API 提供的令牌价格远低于 Claude API。Anthropic 之所以将 Claude Code 定价得更低，推测是为了给用户折扣，鼓励他们使用 Claude Code 而非 OpenCode 等竞争产品。
> 
> 看起来，直到最近，OpenCode 试图通过在其平台内提供“插件”来规避 Anthropic 的要求，这些插件允许用户在 OpenCode 中使用他们的 Claude Code 订阅。这个拉取请求（PR）在 [0][1] 处提到了这一点：
> 
> > 存在一些插件，允许你在 OpenCode 中使用你的 Claude Pro/Max 模型。Anthropic 明确禁止这种行为。
> 
> > 旧版本的 OpenCode 捆绑了这些插件，但从 1.3.0 版本起不再如此。
> 
> 这个 PR 似乎是对 Anthropic 威胁采取法律行动的回应，如果 OpenCode 继续使用内部的 Claude Code API。
>
